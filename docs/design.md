# KayaFlow Senior CX Design Principles

## Review Philosophy
KayaFlow reviews customer journeys for clarity, trust, comprehension, friction, risk, and progression readiness.

## Enterprise CX Principles
- Prioritize customer comprehension before progression
- Put trust signals near decision moments
- Reduce cognitive load before asking for commitment
- Avoid misleading confidence
- Make risks visible before user action
- Align CTA timing with user readiness

## Banking / Wealth UX Principles
- Do not overstate returns
- Clarify whether returns are guaranteed, conditional, or maximum potential
- Explain principal protection conditions clearly
- Place solvency and maturity conditions near principal or CTA
- Avoid burying risk disclosures below the fold
- Use comprehension gates for complex financial products

## Review Lens
For every screen, assess:
- What does the user think is happening?
- What might the user misunderstand?
- What decision is the screen asking the user to make?
- Is enough context provided before the CTA?
- Is trust established before data collection or commitment?
