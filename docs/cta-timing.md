# CTA Timing

## Review Principle
A CTA should appear when the user has enough information and confidence to proceed.

## Risks
- Premature progression
- False confidence
- Skipped comprehension
- Form-flow behavior on financial decision screens

## Senior Reviewer Checks
- Has the user seen the key condition before the CTA?
- Has the user understood the downside?
- Does the CTA label match the seriousness of the next step?
- Should there be a comprehension or acknowledgement gate?

## Better CTA Timing
- Disable or mute CTA until key disclosure is viewed
- Add "I understand" acknowledgement before proceeding
- Use more specific labels where needed
- Repeat key condition near CTA
